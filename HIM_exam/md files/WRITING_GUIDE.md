# 1. 집필 목표

설명은 국가시험의 정답을 맞히는 데 충분해야 하며, 동시에 실제 병원에서 왜 필요한지 이해할 수 있어야 한다. 단순 요약집처럼 정의만 나열하지 않는다.

# 2. 기본 설명 순서

1. **WHY:** 왜 이 개념이 필요한가
2. **WHAT:** 정확한 정의와 범위
3. **HOW:** 실제 적용 과정
4. **EXAMPLE:** 병원 또는 기록 사례
5. **EXAM:** 시험에서 묻는 방식
6. **REVIEW:** 핵심 요약과 점검

# 3. 문체

- 한국어 평서문을 기본으로 한다.
- 한 문장은 가능한 25~35어절을 넘기지 않는다.
- 한 문단에는 하나의 중심 생각을 둔다.
- 불필요한 수사, 과장, 홍보 문구를 피한다.
- “매우 중요하다” 대신 무엇 때문에 중요한지 설명한다.
- 전문용어를 처음 사용할 때 한국어, 영어, 약어를 함께 제시한다.
- 이후에는 가장 일반적인 표기를 사용한다.

예:

> 전자의무기록(Electronic Medical Record, EMR)은 한 의료기관 내부에서 작성·관리되는 전자 형태의 진료기록이다.

# 4. 정의 작성 규칙

좋은 정의는 상위 개념, 핵심 기능, 범위를 포함한다.

나쁜 예:

> EMR은 전자의무기록이다.

좋은 예:

> 전자의무기록(EMR)은 의료기관에서 생성되는 환자의 진료정보를 전자적으로 작성·저장·조회·관리하는 기록체계이다.

# 5. 비교 설명 규칙

비슷한 개념은 서술문만으로 설명하지 않고 최소 2개 이상의 비교 축을 사용한다.

예:

| 구분 | EMR | EHR | PHR |
|---|---|---|---|
| 관리 주체 | 의료기관 | 여러 기관의 연계체계 | 개인 |
| 범위 | 기관 내부 | 기관 간 연속성 | 개인 건강관리 |
| 핵심 키워드 | 진료기록 | 상호운용성 | 자기관리 |

# 6. 법령 집필 규칙

- 법률명, 조문, 기준일을 기록한다.
- 조문을 장문 인용하지 않고 학습자 언어로 설명한다.
- 법적 예외는 본문과 분리해 표시한다.
- 개정 가능성이 큰 수치와 기간은 `as_of_date`를 표시한다.
- 법령과 실무 관행을 혼동하지 않는다.
- “항상”, “절대” 같은 표현은 법적 예외가 없는지 확인한 뒤 사용한다.

# 7. 분류·코드 집필 규칙

- 적용한 ICD/KCD 및 의료행위 분류 버전을 명시한다.
- 색인 접근과 내용예시표 확인 과정을 분리한다.
- 코드 자체보다 선정 논리를 설명한다.
- 실제 시험용 예시는 과도하게 복잡하지 않게 단계적으로 제시한다.
- 코드가 바뀔 가능성이 있으면 정적 본문보다 데이터 파일로 관리한다.

# 8. 통계 집필 규칙

모든 공식은 다음을 포함한다.

1. 지표의 의미
2. 분자
3. 분모
4. 단위
5. 해석
6. 계산 예시
7. 자주 하는 오류

예:

```text
병상이용률
- 의미: 가동 가능한 병상이 실제로 사용된 정도
- 분자: 재원환자 연인원
- 분모: 가동병상일수
- 단위: %
```

# 9. 사례 작성 규칙

- 합성 사례를 사용한다.
- 이름, 생년월일, 주소, 등록번호 등 식별정보를 포함하지 않는다.
- 사례에 필요한 정보만 제공한다.
- 진단 과정과 기록 과정을 혼동하지 않는다.
- 정답을 위해 필요한 단서를 본문에 포함한다.
- 불필요하게 희귀한 질환이나 지나치게 복잡한 임상 추론을 피한다.

# 10. 박스형 콘텐츠

## WHY

개념의 필요성과 배경

## HOW

업무 절차 또는 적용 방법

## LAW

법적 근거와 기준일

## TIP

문제풀이 또는 실무 팁

## CAUTION

혼동하기 쉬운 내용

## AI NOTE

디지털 헬스·AI 환경에서의 변화와 한계

박스가 본문을 반복하지 않도록 한다.

# 11. 국가시험 포인트

시험 포인트에는 다음 중 하나 이상을 명시한다.

- 정의를 바꾸어 출제
- 유사개념 비교
- 순서 배열
- 사례 적용
- 공식 선택
- 분모·분자 판단
- 법령의 주체·의무·예외
- 코드 선정 논리

근거 없이 “빈출”이라고 표시하지 않는다. 빈도 데이터가 없으면 `출제 가능 포인트`라고 쓴다.

# 12. 요약 규칙

요약은 본문을 다시 길게 쓰는 것이 아니다.

- 핵심 정의 1개
- 반드시 구분할 개념 2~4개
- 절차 또는 공식
- 대표 함정
- 다음 Topic과의 연결

# 13. 검토 체크리스트

- [ ] 학습목표와 본문이 일치하는가
- [ ] 용어 정의가 GLOSSARY와 일치하는가
- [ ] 사실과 의견이 구분되는가
- [ ] 법령·표준의 기준일이 있는가
- [ ] 비교표의 기준이 평행한가
- [ ] 사례에 정답 단서가 충분한가
- [ ] 문제의 정답이 본문과 출처로 검증되는가
- [ ] 불필요한 중복이 없는가
- [ ] 의료행위와 정보관리 업무를 혼동하지 않는가
- [ ] 실제 환자 개인정보가 없는가
