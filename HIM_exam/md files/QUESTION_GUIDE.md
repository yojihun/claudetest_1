# 1. 목적

문제는 암기량을 늘리는 도구가 아니라 학습목표 달성 여부를 확인하고 오개념을 교정하는 도구이다.

# 2. Topic별 권장 문항 구성

Topic의 성격에 따라 조정하되 기본 권장량은 다음과 같다.

- OX 3~5문항
- 객관식 3~5문항
- 단답형 또는 계산형 1~2문항
- 사례형 1문항
- 플래시카드 3~8개

모든 Topic에 동일한 수를 억지로 채우지 않는다.

# 3. 인지 수준

| 수준 | 설명 | 예 |
|---|---|---|
| 기억 | 정의·용어·공식 회상 | FHIR의 의미 |
| 이해 | 차이·목적 설명 | EMR과 EHR 비교 |
| 적용 | 사례에 개념 적용 | 무단 열람 사례 판단 |
| 분석 | 여러 기록·수치 비교 | 주진단 후보 분석 |
| 평가 | 가장 적절한 조치 선택 | 개인정보 제공 여부 |
| 생성 | 앱에서는 제한적으로 사용 | 기록 개선안 작성 |

문제은행은 기억형에 편중되지 않도록 한다.

# 4. 객관식 작성 원칙

- 질문은 하나의 명확한 과제를 제시한다.
- 선택지는 문법적·논리적으로 평행하게 작성한다.
- 정답만 유난히 길거나 구체적이지 않게 한다.
- “모두”, “항상”, “절대”를 함정으로 남용하지 않는다.
- 오답은 실제 오개념에서 만든다.
- 복수 정답 가능성을 검토한다.
- 부정형 질문은 필요한 경우에만 사용하고 부정어를 강조한다.
- 선택지 간 포함 관계가 생기지 않도록 한다.

# 5. OX 작성 원칙

- 한 문장에 하나의 판단만 포함한다.
- 일부는 맞고 일부는 틀린 복합 문장을 피한다.
- 단순 단어 치환만 반복하지 않는다.
- 틀린 문항은 어떤 부분이 틀렸는지 해설한다.

# 6. 계산형 작성 원칙

- 필요한 수치를 모두 제공한다.
- 단위를 명시한다.
- 반올림 규칙을 제시한다.
- 공식 암기뿐 아니라 분모·분자 해석을 평가한다.
- 풀이과정을 단계별로 제공한다.
- 잘못된 공식을 선택했을 때의 오류도 설명한다.

# 7. 사례형 작성 원칙

사례에는 다음 구조를 권장한다.

1. 의료기관 상황
2. 필요한 기록 또는 수치
3. 수행해야 할 과제
4. 정답 판단에 필요한 단서
5. 불필요한 임상정보 최소화

# 8. 해설 구조

모든 객관식 해설은 다음을 포함한다.

```text
정답:
핵심 근거:
정답 선택지가 맞는 이유:
각 오답이 틀린 이유:
관련 Topic:
출처:
```

# 9. 난이도

## Basic

정의와 대표 특징을 직접 묻는다.

## Intermediate

두 개 이상의 개념을 구분하거나 간단한 사례에 적용한다.

## Advanced

여러 기록, 예외, 법령, 분류 또는 계산 단계를 통합한다.

난이도는 문장이 긴지 여부가 아니라 해결에 필요한 사고 단계로 판단한다.

# 10. 오답 설계

좋은 오답은 다음에서 나온다.

- 유사 용어 혼동
- 절차 순서 오류
- 주체와 책임의 혼동
- 분모와 분자 교환
- 기관 내부와 기관 간 범위 혼동
- 기록관리와 직접 진료행위 혼동
- 현재 법령과 과거 법령 혼동
- 색인과 내용예시표 확인 순서 혼동

# 11. 문제 검증

문제는 최소 두 단계로 검토한다.

## 내용 검토

- 정답의 사실성
- 법령·분류 버전
- 계산 결과
- 의료용어 정확성

## 문항 검토

- 복수 정답 여부
- 힌트가 되는 표현
- 불필요한 난해성
- 선택지 평행성
- 목표 인지 수준

# 12. 중복 탐지

문제의 중복은 문장 유사도만으로 판단하지 않는다. 다음이 같으면 개념적 중복 가능성이 높다.

- 동일 Topic
- 동일 학습목표
- 동일 정답 근거
- 동일 오답 패턴
- 동일 사례 구조

# 13. 시험 블루프린트

모의고사 생성 시 다음을 기록한다.

```yaml
exam_id:

topic_distribution:
question_type_distribution:
difficulty_distribution:
time_limit:
randomization_seed:
```

# 14. 금지사항

- 근거 없는 “기출” 표기
- 실제 공식문항을 허가 없이 대량 복제
- 정답이 설명과 일치하지 않는 문제
- 법령 기준일이 없는 법규 문제
- 코드 버전이 없는 분류 문제
- 계산 과정 없이 숫자만 제시한 해설
- 학생을 속이는 문법적 함정
