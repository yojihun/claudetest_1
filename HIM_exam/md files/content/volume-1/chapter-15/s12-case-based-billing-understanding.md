# 사례 기반 청구 이해

## 학습목표

1. Chapter 15에서 다룬 개념(급여·비급여, DRG, 심사·평가, 삭감)을 하나의 통합 청구 사례에 적용할 수 있다.

## 한 문장 정의

이 Topic은 Chapter 15(건강보험)에서 다룬 개념들을 하나의 통합 사례에 적용해 보는 사례 기반 복습 Topic이다.

## 왜 중요한가

실제 청구·심사 문제는 여러 개념(급여 기준, 분류 코드, 기록 완결성)이 얽혀 나타나는 경우가 많다.

## 선수 개념

- [삭감과 기록](s10-claim-denials-and-documentation.md)
- [HIRA·NHIS의 역할](s11-hira-nhis-roles.md)

## 핵심 개념

사례 분석 절차:

1. 급여·비급여 구분 확인
2. 분류 코드의 정확성 확인
3. 기록의 뒷받침 여부 확인(CDI 관점)
4. 삭감 발생 시 원인 분석(기준 위반 vs. 기록 미비 vs. 코드 오류)

## 상세 설명

아래 사례를 위 절차로 분석해본다.

> **사례:** 한 병원이 특정 수술(포괄수가제 적용 대상)을 시행하고 청구했으나, HIRA 심사에서 일부 금액이 삭감되었다. 조사 결과, 수술 자체는 적절했으나 수술 후 발생한 합병증에 대한 기록이 퇴원요약에 명확히 기재되어 있지 않아 DRG 분류 시 합병증 유무가 반영되지 못했다.

## 비교표 또는 흐름

| 단계 | 이 사례에서의 분석 |
|---|---|
| 급여·비급여 확인 | 급여 대상 수술(포괄수가제 적용) |
| 분류 코드 확인 | 합병증 관련 코드가 기록 미비로 누락 |
| 기록 확인 | 퇴원요약에 합병증 기재 부족 |
| 삭감 원인 분석 | 기록 미비로 인한 DRG 분류 오류 |

## 실무 사례

(본 Topic 자체가 통합 사례이므로 별도 사례를 추가하지 않는다.)

## 국가시험 포인트

- 삭감의 표면적 결과(금액 삭감)와 근본 원인(기록 미비)을 구분하는 능력
- 포괄수가제(DRG) 환경에서 기록의 완결성이 왜 재정적으로도 중요한지 이해했는지 확인하는 문제

## 자주 하는 실수

- 삭감을 청구 담당자의 실수로만 단정하고 근본 원인(임상 기록의 완결성)을 확인하지 않는 것

## AI Note

AI 기반 CDI 도구가 퇴원요약 작성 시 합병증 기재 누락을 자동으로 알릴 수 있으나, 최종 확인과 기재는 담당 의료진의 책임이다.

## 핵심 요약

- 청구 삭감 사례는 급여 구분, 코드 정확성, 기록 완결성을 함께 확인해 분석해야 한다.
- 포괄수가제 환경에서는 기록의 완결성이 재정적 결과에 직접 영향을 준다.
