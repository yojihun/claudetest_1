# 질병 및 의료행위 분류

## 학습목표

1. 분류 직무의 핵심 절차(진단명 확인 → 코드 부여 → 검증)를 설명할 수 있다.
2. 분류 직무가 청구·통계와 어떻게 연결되는지 설명할 수 있다.

## 한 문장 정의

질병 및 의료행위 분류 직무는 확정된 진단명과 시행된 의료행위를 표준 분류체계(KCD, 의료행위 분류 등)의 코드로 정확히 변환하는 업무이다.

## 왜 중요한가

부여된 코드는 청구, 통계, 질 지표 산출의 기초 자료가 되므로, 코드 하나의 오류가 여러 후속 업무에 영향을 줄 수 있다. 이 영역의 상세 내용(KCD 구조, 코딩 절차 등)은 Volume III에서 깊이 다룬다.

## 선수 개념

- [직무 개요](s01-job-overview.md)

## 핵심 개념

- 진단명·의료행위명 확인(수술기록, 경과기록 등 원본 확인)
- 분류체계 적용(색인 확인 → 내용예시표 확인의 순서 준수)
- 코드 검증(주진단·기타진단 선정 기준 확인)
- 버전 관리(적용한 분류체계 버전 기록)

## 상세 설명

분류 직무는 코드를 "암기"하는 일이 아니라 "선정 논리"를 적용하는 일이다. 동일한 진단이라도 기록에 어떤 정보가 있는지에 따라 다른 코드가 부여될 수 있으므로, 코더는 항상 원본 기록(수술기록, 병리 결과 등)을 근거로 판단해야 한다. 이 절차의 상세 내용(색인·내용예시표, 주진단 선정 기준 등)은 Volume III Part 2에서 별도로 다룬다.

## 비교표 또는 흐름

| 단계 | 내용 |
|---|---|
| 1. 원본 확인 | 수술기록, 병리결과, 경과기록에서 확정 진단·시행 행위 확인 |
| 2. 색인 확인 | 분류체계의 알파벳 색인에서 후보 코드 탐색 |
| 3. 내용예시표 확인 | 후보 코드의 정확한 적용 범위·포함제외지시 확인 |
| 4. 코드 확정 | 주진단·기타진단 등 역할 구분 후 코드 확정 |
| 5. 검증 | 기록과 코드의 일치 여부 재확인 |

## 실무 사례

한 코더가 수술 후 진단명만 보고 코드를 부여했다가, 실제 병리 결과 보고서에 기재된 더 구체적인 진단명을 확인하지 않아 코드가 부정확했던 사례가 있다. 이는 분류 직무에서 원본 확인이 왜 중요한지 보여준다.

## 국가시험 포인트

- 분류 직무의 절차 순서를 배열하는 문제
- 분류 직무의 오류가 어떤 후속 업무(청구, 통계)에 영향을 주는지 묻는 문제

## 자주 하는 실수

- 색인만 확인하고 내용예시표를 확인하지 않은 채 코드를 확정하는 것
- 코드 부여를 암기의 문제로만 여기고 원본 기록 확인을 생략하는 것

## 관련 법령·표준

적용한 KCD·의료행위 분류의 정확한 버전과 기준일은 Volume III Part 2에서 구체적으로 다룬다.

## AI Note

AI 기반 자동 코딩 추천 도구가 확산되고 있으나, 추천된 코드가 실제 기록과 일치하는지 최종 검증하는 것은 코더의 책임이다.

## 핵심 요약

- 분류 직무는 원본 확인 → 색인 확인 → 내용예시표 확인 → 코드 확정 → 검증의 절차를 따른다.
- 코드 오류는 청구·통계 등 후속 업무에 영향을 준다.

## 플래시카드

2. Q. 코드 부여의 근거가 되어야 하는 것은? / A. 원본 기록(수술기록, 병리결과 등)
