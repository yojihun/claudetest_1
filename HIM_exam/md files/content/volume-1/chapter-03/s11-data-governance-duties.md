# 데이터 거버넌스

## 학습목표

1. 데이터 거버넌스와 데이터 품질관리의 차이를 설명할 수 있다.
2. 데이터 거버넌스의 핵심 구성 요소(정책, 책임, 표준)를 나열할 수 있다.

## 한 문장 정의

데이터 거버넌스는 조직 전체 차원에서 데이터를 누가 소유·관리·책임지는지, 어떤 규칙에 따라 다룰지를 정하는 정책과 체계이다.

## 왜 중요한가

[데이터 품질관리](s08-data-quality-management-duties.md)가 "실무 차원의 점검·개선"이라면, 데이터 거버넌스는 "조직 차원의 규칙과 책임 체계"다. 거버넌스 없이는 품질관리 활동도 일관되게 지속되기 어렵다.

## 선수 개념

- [데이터 품질관리](s08-data-quality-management-duties.md)

## 핵심 개념

- 데이터 정책: 데이터를 어떻게 수집·관리·활용할지에 대한 조직 차원의 규칙
- 데이터 책임자(스튜어드십): 특정 데이터 영역에 대한 책임 소재 지정
- 표준 관리: 조직 전체가 따라야 할 데이터 정의·표준 확정
- 위원회·의사결정 체계: 데이터 관련 의사결정을 조율하는 조직 구조

## 상세 설명

데이터 품질관리가 "이 항목이 왜 비어 있는가"를 다루는 실무 활동이라면, 데이터 거버넌스는 "이 항목은 누가 책임지고, 어떤 정의를 따라야 하는가"를 정하는 상위 체계다. 예를 들어 "환자 체중"이라는 항목을 어느 부서가 어떤 단위(kg)로 입력할지에 대한 조직 차원의 규칙이 없다면, 개별 부서가 각자 다른 방식으로 입력해 [일관성](../chapter-01/s15-consistency-reliability-integrity.md) 문제가 반복해서 발생한다. 거버넌스는 이런 반복적 문제의 근본 원인(규칙 부재)을 해결한다.

## 비교표 또는 흐름

| 구분 | 데이터 품질관리 | 데이터 거버넌스 |
|---|---|---|
| 층위 | 실무 차원 | 조직 차원 |
| 초점 | 개별 오류 탐지·개선 | 정책·책임·표준 수립 |
| 예시 활동 | 중복 데이터 정리 | "환자 체중은 kg 단위로, OO부서가 입력 책임" 규칙 수립 |

## 실무 사례

한 병원에서 여러 부서가 서로 다른 방식으로 환자 식별 정보를 입력해 반복적으로 중복 등록 문제가 발생했는데, 데이터 품질관리 차원의 개별 수정만으로는 근본적으로 해결되지 않다가, 조직 차원의 데이터 거버넌스 위원회가 입력 규칙과 책임 부서를 명확히 정하면서 문제가 크게 줄어든 사례가 있다.

## 국가시험 포인트

- 데이터 품질관리와 데이터 거버넌스의 층위 차이를 구분하는 문제
- 반복되는 데이터 문제의 근본 원인이 거버넌스 부재임을 판단하는 사례형 문제

## 자주 하는 실수

- 데이터 거버넌스를 데이터 품질관리와 같은 활동으로 혼동하는 것

## 관련 법령·표준

이 Topic은 조직 관리 개념으로 특정 법령에 근거하지 않는다.

## AI Note

AI 활용이 늘어날수록 "AI가 어떤 데이터를 학습에 사용할 수 있는가"를 정하는 것도 데이터 거버넌스의 새로운 영역이 되고 있다.

## 핵심 요약

- 데이터 거버넌스는 조직 차원의 정책·책임·표준 수립 체계다.
- 반복되는 개별 오류의 근본 원인이 규칙 부재라면 거버넌스 차원의 해결이 필요하다.
