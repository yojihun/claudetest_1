# 데이터 결합

## 학습목표

1. 서로 다른 기관의 가명 데이터를 결합할 때 특별한 절차가 필요한 이유를 설명할 수 있다.
2. 데이터 결합이 재식별 위험을 높일 수 있는 이유를 설명할 수 있다.

## 한 문장 정의

데이터 결합은 서로 다른 기관(병원, 통계청, 건강보험공단 등)이 보유한 가명 처리된 데이터를 특정 절차를 거쳐 하나로 합쳐 분석하는 것을 말한다.

## 왜 중요한가

각각은 [가명처리와 연구](s11-pseudonymization-for-research.md)에서 다룬 것처럼 재식별이 어렵더라도, 여러 출처의 데이터를 결합하면 조합을 통해 개인이 재식별될 위험이 커질 수 있다.

## 선수 개념

- [가명처리와 연구](s11-pseudonymization-for-research.md)

## 핵심 개념

- 결합 전문기관: 데이터 결합은 통상 지정된 전문기관을 통해서만 이루어지도록 제한되는 경우가 많음
- 결합 후 재평가: 결합된 데이터가 재식별 위험이 없는지 다시 평가
- 목적 제한: 결합된 데이터는 애초에 승인된 연구 목적 범위 내에서만 사용

## 상세 설명

한 병원의 가명 데이터만으로는 개인을 특정하기 어렵더라도, 여기에 다른 기관의 데이터(예: 특정 지역의 건강보험 청구 데이터)를 결합하면 두 데이터셋의 교집합을 통해 특정 개인이 좁혀질 위험이 생긴다. 이 때문에 데이터 결합은 개별 기관이 임의로 수행하지 않고, 지정된 절차와 기관을 통해서만 이루어지도록 관리되는 경우가 많다.

## 비교표 또는 흐름

| 단계 | 위험 관리 |
|---|---|
| 결합 전 | 각 데이터셋의 가명처리 상태 확인 |
| 결합 절차 | 지정 전문기관을 통한 결합 |
| 결합 후 | 재식별 위험 재평가 |

## 실무 사례

두 기관이 각자 가명처리된 데이터를 보유하고 있었지만, 결합 시 재식별 위험이 있다고 판단되어 지정된 결합 전문기관을 통해 안전하게 결합한 사례가 있다.

## 국가시험 포인트

- 데이터 결합이 재식별 위험을 높일 수 있다는 점을 확인하는 문제

## 자주 하는 실수

- 각 데이터셋이 개별적으로 안전하면 결합해도 안전하다고 오해하는 것
