# 코드 체계 관리

## 학습목표

1. 병원이 사용하는 여러 코드 체계(KCD, LOINC, SNOMED CT 등)를 관리하는 절차를 설명할 수 있다.
2. 코드 체계 개정 시 병원이 해야 할 조치를 설명할 수 있다.

## 한 문장 정의

코드 체계 관리는 병원이 사용하는 여러 표준 코드 체계의 버전을 추적하고, 개정 시 시스템과 매핑 테이블을 함께 갱신하는 지속적 관리 활동이다.

## 왜 중요한가

[KCD](s12-kcd.md)나 다른 표준이 개정될 때마다 시스템을 갱신하지 않으면, 오래된 코드로 계속 분류하는 오류가 누적된다.

## 선수 개념

- [용어 매핑](s13-terminology-mapping.md)

## 핵심 개념

- 버전 추적: 현재 병원이 사용 중인 각 코드 체계의 버전을 기록
- 개정 반영: 새 버전이 발표되면 시스템, 매핑 테이블, 교육 자료를 함께 갱신
- 이행 기간 관리: 신버전 적용 시점과 구버전 데이터의 처리 방침 결정

## 상세 설명

코드 체계는 한 번 도입하고 끝나는 것이 아니라, 지속적으로 개정된다. 병원은 새 버전이 나올 때마다 "언제부터 신버전을 적용할지", "기존에 구버전으로 입력된 데이터는 어떻게 표시할지"를 결정해야 한다. 이 결정이 없으면 같은 병원 안에서도 신버전과 구버전 코드가 혼재해 통계 집계 시 혼란이 생길 수 있다.

## 비교표 또는 흐름

| 관리 활동 | 내용 |
|---|---|
| 버전 추적 | 현재 사용 버전 기록 |
| 개정 반영 | 시스템·매핑 테이블·교육 자료 갱신 |
| 이행 기간 관리 | 신버전 적용 시점, 구버전 데이터 처리 방침 결정 |

## 실무 사례

한 병원에서 KCD 신버전이 발표되었지만 시스템 갱신이 지연되어, 일부 코더는 구버전으로 신버전 시행 이후에도 계속 코딩한 사례가 있다. 이는 개정 반영 지연이 실제로 오류를 만들 수 있음을 보여준다.

## 국가시험 포인트

- 코드 체계 개정 시 병원이 취해야 할 조치를 확인하는 문제

## 자주 하는 실수

- 코드 체계를 한 번 도입하면 별도 관리 없이 계속 사용해도 된다고 오해하는 것

## 관련 법령·표준

각 코드 체계의 개정 공지는 관리 주체(WHO, 국내 관계 기관 등)의 공식 발표를 통해 확인해야 한다.

## AI Note

시스템이 코드 체계 버전을 자동으로 알림 처리할 수 있으나, 실제 반영과 이행 기간 결정은 담당자의 관리가 필요하다.

## 핵심 요약

- 코드 체계 관리는 버전 추적, 개정 반영, 이행 기간 관리로 구성된다.
- 개정 반영이 지연되면 신·구 버전이 혼재해 통계 오류로 이어질 수 있다.
