# 사례형 통합 코딩

## 학습목표

1. 이 Part에서 배운 여러 코딩 원칙([주진단 선택](s03-principal-diagnosis-selection.md), [기타진단](s04-other-diagnosis-symptom-vs-confirmed.md), [의료행위 분류](s10-procedure-classification-overview.md) 등)을 하나의 실제 사례에 종합적으로 적용할 수 있다.
2. 사례형 코딩 문제를 풀 때의 단계별 접근 순서를 설명할 수 있다.

## 한 문장 정의

이 Topic은 이 Part에서 배운 개별 코딩 원칙들을 하나의 통합된 임상 사례에 적용해, 주진단·기타진단·의료행위를 모두 정확히 코딩하는 종합 연습을 다룬다.

## 왜 중요한가

실제 국가시험과 실무에서는 개별 개념을 따로 묻기보다, 하나의 사례 안에서 여러 원칙을 동시에 적용해야 하는 경우가 대부분이므로 통합적 사고 훈련이 필수적이다.

## 선수 개념

- [코딩 감사](s13-coding-audit.md)

## 핵심 개념

- 사례형 코딩의 단계: (1) 입원 당시 의심 진단과 퇴원 시 확정 진단 구분 → (2) [주진단](s03-principal-diagnosis-selection.md) 선택 → (3) [기타진단](s04-other-diagnosis-symptom-vs-confirmed.md)(합병증 포함) 식별 → (4) 시행된 [의료행위](s10-procedure-classification-overview.md) 코딩(접근 방식 포함) → (5) 필요시 [외인분류](s06-external-cause-injury-classification.md) 추가
- 종합 검토: 부여된 코드가 전체 의무기록과 일관되는지 최종 확인([코딩 감사](s13-coding-audit.md) 관점 적용)

## 상세 설명

다음 사례로 통합 코딩 절차를 연습한다: "환자가 계단에서 낙상해 복통을 주소로 입원. 검사 결과 급성 충수염 및 국소 복막염 확인. 복강경 충수절제술 시행, 수술 중 출혈로 개복 전환." 이 사례에서는 (1) 입원 당시 복통(의심)이 아닌 급성 충수염(확정)을 주진단으로, (2) 국소 복막염을 합병증을 포함한 기타진단으로, (3) 복강경으로 시작했으나 개복으로 전환된 수술을 정확한 접근 방식으로 반영한 의료행위 코드로, (4) 낙상이 이 입원과 직접 관련이 없다면 이 사례에서는 외인분류가 적용되지 않는다는 점까지 종합적으로 판단해야 한다.

## 비교표 또는 흐름

| 단계 | 이 사례에서의 적용 |
|---|---|
| 주진단 선택 | 급성 충수염(확정 진단) |
| 기타진단 | 국소 복막염(합병증) |
| 의료행위 코딩 | 충수절제술(개복 전환 반영) |

## 실무 사례

코딩 교육 과정에서 이런 복합 사례를 반복적으로 연습시켜, 실제 업무에서 여러 정보가 뒤섞인 의무기록을 마주했을 때도 체계적으로 정보를 분리·정리하는 능력을 기르도록 한다.

## 국가시험 포인트

- 복합 임상 사례에서 여러 코딩 원칙을 순서대로 적용해 종합적으로 코딩하는 문제

## 자주 하는 실수

- 사례의 일부 정보(예: 접근 방식 전환, 합병증)를 놓치고 가장 눈에 띄는 정보만으로 코딩을 마무리하는 것

## 관련 법령·표준

이 사례의 구체적 코드는 이 세션에서 원문 대조하지 않았으므로, 실제 코딩 시 공식 분류집을 통해 확정해야 한다.
