# 알파벳 색인·내용예시표·포함제외지시·기호약어

## 학습목표

1. 알파벳 색인과 내용예시표(본문표)의 역할 차이를 설명할 수 있다.
2. 포함 지시와 제외 지시가 코드 선택에 미치는 영향을 설명할 수 있다.

## 한 문장 정의

분류집은 진단명으로 코드 후보를 찾는 알파벳 색인과, 찾은 코드의 정확한 정의·범위를 확인하는 내용예시표(본문표)로 구성되며, 각 코드에는 포함·제외 지시와 기호·약어가 함께 표기된다.

## 왜 중요한가

알파벳 색인에서 찾은 코드를 확인 없이 그대로 사용하면, 실제로는 그 코드에서 제외되어야 할 진단을 잘못 분류하는 오류가 발생할 수 있다.

## 선수 개념

- [분류의 목적과 ICD·KCD의 역사와 구조](s01-classification-purpose-icd-kcd-history.md)

## 핵심 개념

- 알파벳 색인(Alphabetic Index): 진단명을 알파벳(또는 가나다) 순으로 나열해 후보 코드를 찾는 부분
- 내용예시표/본문표(Tabular List): 코드 순서대로 나열되어 각 코드의 정확한 정의·범위를 보여주는 부분
- 포함 지시(Includes): 해당 코드에 포함되는 진단·상태를 명시
- 제외 지시(Excludes): 해당 코드에서 제외되어 다른 코드로 분류해야 하는 진단·상태를 명시

## 상세 설명

코딩 작업은 일반적으로 알파벳 색인에서 진단명으로 후보 코드를 찾는 것에서 시작하지만, 이 후보 코드를 그대로 사용해서는 안 되며 반드시 내용예시표(본문표)에서 해당 코드의 정확한 정의와 포함·제외 지시를 확인하는 절차를 거쳐야 한다. 제외 지시는 특히 중요한데, 겉보기에 비슷해 보이는 진단이 실제로는 완전히 다른 코드로 분류되어야 함을 알려주는 경고와 같은 역할을 한다. 이 지시들을 무시하고 알파벳 색인의 첫 코드만 사용하면 코딩 오류로 이어질 수 있다.

## 비교표 또는 흐름

| 단계 | 부분 | 역할 |
|---|---|---|
| 1단계 | 알파벳 색인 | 진단명으로 후보 코드 탐색 |
| 2단계 | 내용예시표(본문표) | 후보 코드의 정확한 정의·포함제외지시 확인 |

## 실무 사례

코딩 담당자가 알파벳 색인에서 특정 진단명의 코드 후보를 찾은 후, 본문표에서 "제외: ○○"라는 지시를 확인해 실제로는 다른 코드가 더 적절함을 발견하고 코드를 수정하는 경우가 흔히 있다.

## 국가시험 포인트

- 알파벳 색인과 내용예시표의 역할 차이, 제외 지시를 무시했을 때의 위험을 확인하는 문제

## 자주 하는 실수

- 알파벳 색인에서 찾은 코드를 본문표에서 재확인하지 않고 그대로 사용하는 것

## 관련 법령·표준

이 Topic이 다루는 색인·본문표 구조는 ICD/KCD의 일반적 편제 원칙이며, 구체적 코드 번호는 이 세션에서 원문 대조하지 않았으므로 공식 분류집으로 확인이 필요하다.
